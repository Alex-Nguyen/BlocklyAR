Tutorial 1
In the first tutorial, you will be asked to initialize the Augmented Reality scene. Click on the A-Frame and drag the AR SCENE to the workspace then click the Run Program button. The camera in the visualization area will be triggered
Tutorial 2
In the second step, you will be asked to put the marker in the scene and associated model once the marker is detected by the camera. Go go A-FRAME/Markers => Drag the marker block inside of the AR scene then drag the A-FRAME/Sample models/Cube inside of the marker. Run the program, after this step you should be able to see the red cube shown on top of the marker. The default marker is Hiro which can be downloaded via this link. Place the marker in front of the camera

The layout structure of the tool is appropriated
Tutorial 3
In this step, you will be asked to remove the existing cube model and change the marker type from preset to pattern. To change the cube, click on the cube then hit Delete button (or drag it to the trash icon), go to Sample models and drag the GLTF model inside the marker. To change the marker type, delete it, go to Markers then drag the Pattern block and attach it to the old preset pattern

The layout structure of the tool is appropriated
Tutorial 4
In this step, you are asked to add another marker to the scene. Let add another marker, its content (GLTF model) then set the pattern of the first marker to letter A, the pattern of the second marker to letter B. Run the program, you should be able to see two models sitting on top of the two markers (letter A and letter B)
Tutorial 5
Let add a button into the scene to control the model. Drag the "Nav-bar" block to the workspace and position it on top of the AR-SCENE block. Run the program, you should be able to see Button 1 appears on top of the visualization's window.
Tutorial 6
Let add three more buttons to the scene. (Hint: Drag the button block inside the nav-bar) and change the names of the four buttons to S1, S2, S3, S4 respectively.
Tutorial 7
This step will init a controller to buttons that you've created in the previous step. Drag the Script block to the workspace on top of the Nav-bar block. You can check if you've done correctly by going to step 9
Tutorial 8
Since the control block must know which button it refers to, we have to create a unique id for each button. Go to Variables, click on create variables then create 4 variables and named it stage1, stage2, stage3, stage4 respectively. Then set the button ID to its associated variable.
Tutorial 9
This step will introduce you a simple behavior when the button 1 is clicked. It will display a "Hello world" message on the screen. Select button id as stage1 in the set button block, then attach the Behaviour block to the Button block. You also need to embed a Display message block inside the behavior. Run the program and click on the button S1, You should be able to see the message "Hello world" overlay on the screen.
Tutorial 10
Can you add more behaviours to button 1? such as play an audio, display the message, move an object and change the animations of a given model???
For a logical programming, it is noted that each object should have an unique id. Try to create an id for each block (e.g., marker's id and model's id)
